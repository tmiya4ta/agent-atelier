# Atelier 操作手順書（ユーザーガイド）

Atelier はブラウザだけで動く A2A / MCP マルチエージェント用のワークベンチです。複数のエージェント窓を並べ、
シナリオ（会話 DSL）で会話を再生できます。本書は **画面の使い方** をまとめた操作手順書です。

- 概要・機能一覧 → [`README.md`](../README.md)
- 設計・データフロー → [`docs/architecture.md`](architecture.md)
- ローカル開発・CH2 デプロイ・ハマりどころ → [`ONBOARDING.md`](../ONBOARDING.md)

![Atelier の起動直後（エージェント未接続のホーム画面）](img/overview-empty.png)

## 0. 起動

ES Modules を使うため `file://` 直開きでは動きません。dev サーバ経由で開きます。

```sh
node server/dev-server.js --port 8000
# → ブラウザで http://127.0.0.1:8000/
```

> CloudHub 2.0 にホストされた版（例: `https://atl2-23fgzd.pnwfdv.jpn-e1.cloudhub.io/`）は
> そのまま URL を開くだけで使えます。

## 1. 画面の構成

![画面の構成：左サイドバー＋ワークスペース（6 つのエージェント窓をタイル表示）＋上部のワークスペースタブ](img/ws-departments.png)

*↑ 保険シナリオを取り込んだ状態。中央に 6 部門のエージェント窓が並び、上部にワークスペースタブ（`保険 — 部門別 / 保険 — Broker`）が見えます。*

- **左サイドバー**：アイコンで `Connections / Catalogs / Authentication / Scripts` を切り替えます
- **中央**：エージェント窓を自由に配置できます（ドラッグで移動、端をドラッグでリサイズ）
- **下部**：Script Panel（シナリオの編集と実行）
- **ワークスペース**：窓のセットを「作業空間」として複数保持できます（上部バーのタブで切り替え）

## 2. エージェントに接続する（New Connection）

1. 左サイドバー **Connections** → **`+ new connection`** をクリック。
2. **protocol** を選ぶ：
   | プロトコル | 用途 |
   |---|---|
   | **A2A** | 会話するエージェント（チャット） |
   | **MCP** | ツールを呼ぶサーバ（tools タブでフォーム実行） |
   | **Slack** | Slack 互換 Web API |
   | **Mock** | 疑似エージェント（後述。実通信なし・デモ用） |
3. **URL** を入力します（A2A は base URL で構いません。Atelier が `/.well-known/agent-card.json` を自動取得します）。
4. （任意）**display name** で表示名、**auth** で認証 identity を指定します。
5. **test** で接続を確認し、**connect** を押すと窓が開きます。

![New Connection ダイアログ：protocol（A2A / Slack / MCP / Mock / OpenAI）、Discovery URL、display name、auth](img/dialog-new-connection.png)

接続すると **Connections 一覧に登録**され、次回からはワンクリックで窓を開けます（接続はブックマークとして保持されます）。
同じエージェントの窓を追加するには、一覧の行の **`+`** を押します。

### 窓のタブ
- **chat**：会話タブです。下の入力欄にメッセージを入力して送信します（A2A は SSE ストリーミングで表示されます）。
- **card**：AgentCard / MCP サーバの情報（能力・スキル一覧）を表示します。
- **debug**：生の RPC フレームを時系列で表示します。各フレームを展開して payload / headers を確認できます。
- **settings**：Discovery URL / 実エンドポイント / 認証 / プロトコルを確認・変更できます。表示名のインライン編集もここで行います。

| chat タブ | debug タブ |
|---|---|
| ![chat タブ：エージェント応答を Markdown で表示](img/window-chat.png) | ![debug タブ：JSON-RPC フレームを時系列表示](img/window-debug.png) |

## 3. Mock（疑似エージェント）を使う

実サーバがなくてもデモできる、**実通信しない疑似エージェント**です。
見た目・挙動は本物の A2A / MCP と同じで、UI 上は「mock」と表示されません（一覧では破線＋`sim` バッジで区別されます）。

1. New Connection で **protocol = Mock** を選択します。
2. **kind** で「装うプロトコル」を選びます：**A2A**（会話）/ **MCP**（ツールサーバ）。
3. URL 欄が **agent name** に変わるので、役割を表す名前（例：`事故受付`）を入力して connect を押します。

mock 窓は、後述のシナリオ（`$>` 行）の応答を再生するために使います。チャットに直接メッセージを打つと、そのエージェントの担当範囲を案内する定型応答が返ります（シナリオに同梱された mock の場合）。

---

## 4. シナリオを読み込む — Import

配布済みのデモシナリオを取り込めます。

1. 上部バーまたはサイドバーの **Import** → **From remote site** → **Repository** を選択します。
2. 一覧から取り込みたいシナリオを選びます：
   - **SCRS — シナリオ A/B**（製造業：品質トラブル横断対応）
   - **保険 — 自動車保険金請求**（保険：6部門の請求処理デモ）
3. **Scenarios only**（既定 ON）を ON にするとシナリオだけを追加します（接続は維持）。OFF にすると接続・窓ごとすべて置き換えます（リロード）。

![Import from repository ダイアログ：Scenarios only チェックボックスと、SCRS / 保険 の 2 シナリオ](img/dialog-import-repository.png)

> **From file…** でローカルの `.json` スナップショットも取り込めます（同じ「Import」ダイアログ）。

取り込むと Scripts にシナリオが追加され、everything 取込の場合は窓とワークスペースも復元されます。

---

## 5. シナリオを実行する — Script Panel

1. 左サイドバーの **Scripts** でシナリオを選びます（Script Panel が下からせり上がります）。
2. **▶ Run** を押すか、エディタにフォーカスした状態で `⌘/Ctrl + Enter` を押すとシナリオを実行します。
3. 実行中は、シナリオの記述に沿って各窓へメッセージが送信され、応答が typewriter 形式で流れます。

![Script Panel：上部に window / command チップ、左に MOCK トグル（自動 ON で点灯）、下にシンタックスハイライト付きエディタ](img/script-panel.png)

*↑ 保険シナリオを開いた状態。参照する窓がすべて mock 接続なので **MOCK** が自動 ON（橙色）になっています。*

### MOCK モード（実通信なし・ローカル応答）
- シナリオの `$>` 行に書いた応答をローカルで再生する機能です。**MOCK ボタン**で ON/OFF を切り替えます。
- **シナリオが参照する窓がすべて mock 接続の場合、MOCK モードは自動的に ON になります**（ボタン操作は不要です）。
  保険シナリオのように全 mock で構成されたデモは、Import して Run するだけで応答が流れます。

### シナリオ（DSL）の書き方
1 行 = 1 命令です。エディタ下部の補完チップからも挿入できます。

| 記法 | 意味 |
|---|---|
| `< 窓名: メッセージ` | その窓へ送信（`${変数}` 展開可） |
| `> 窓名` / `> 窓名 30s` / `> 窓名 30s as 変数` | 応答待ち（タイムアウト指定・応答を変数に保存） |
| `$> 窓名: 応答` | mock 応答（送信と対称）。MOCK モード時に再生。改行は `\n` |
| `sleep 2s` | 一時停止 |
| `clear` / `clear 窓名` | 全窓 / 指定窓のチャットをクリア |
| `# コメント` | コメント行 |

**`$>` を連続して並べると順次表示**：同じ窓宛の `$>` を（間に他の行を挟まず）連続して書くと、1 回の応答の中で**ステップを 1 つずつ順番に**表示します（1 ステップを 1 行で記述できます）。
間に `<` / `>` / `sleep` が入ると、次の送信用の応答として区切られます。

```
< 受付: 事故第一報を起票してください。
> 受付
$> 受付: 受付番号 AC-... を採番しました。
$> 受付: 損害サービスへ引き継ぎました。      # ← 上の行と続けて順次表示される
```

> 実行前にまだ開いていない窓は、登録済みの接続（ブックマーク）から自動的に開かれます。

## 6. ワークスペース（作業空間）

窓のセットを「作業空間」として複数持てます。上部バーのタブで切り替えます。

- 保険シナリオでは **WS1=部門別 6 窓（FA1 用）** / **WS2=Broker 1 窓（FB1 用）** のように分かれています。
- タブの **`+`** で新規作成できます。

## 7. キーボードショートカット

`⌘`（macOS）/ `Ctrl`（Windows）。

| キー | 動作 |
|---|---|
| `⌘1`〜`⌘9` | N 番目の窓にフォーカス |
| `⌘⇧[` / `⌘⇧]` | ワークスペース切替（前 / 次） |
| `⌘⇧K` | Script Panel の開閉 |
| `⌘⏎`（エディタにフォーカス時） | シナリオを実行する |
| `⌘.`（Script Panel 表示時） | シナリオを停止する |
| `⌘W`（エディタにフォーカス時） | 現在のシナリオタブを閉じる |
| `Esc` | ダイアログ / メニューを閉じる |

> 「new connection」「new workspace」は対応する **`+` ボタン**から行います。

---

## 8. 設定の保存・バックアップ

- 接続・カタログ・シナリオ・ワークスペースは **localStorage に自動保存**されます。
- **Export** で JSON スナップショットを書き出せます。**Import** で復元・共有が可能です。
- **secrets（OAuth client_secret / token）は localStorage に保存されず、export にも含まれません**
  （sessionStorage に分離されており、タブを閉じると消えます）。
- **Reset** で全データを消去して再読み込みします。

## 9. デモの流れ（例：保険シナリオ）

1. **Import** → Repository → 「保険 — 自動車保険金請求」を選んで everything で取り込みます。
2. **WS1（部門別 6 窓）** でシナリオ **FA1** を Run します
   → Broker なしで、担当者が 6 部門を順に照会・3 往復ずつ持ち回る「現状の手間」を再現します。
3. **WS2（Broker 1 窓）** でシナリオ **FB1** を Run します
   → 保険オーケストレーターに 1 文を投げるだけで、6 部門を順次処理して統合し、お客様向けご案内文の清書まで肩代わりする「Agent Network の価値」を示します。

全 mock 接続なので、実エンドポイントなしでそのまま再生できます（MOCK 自動 ON）。

保険シナリオを Import したあと、**①まず Broker なし（FA1）** を部門別ワークスペースで実行し「人が 6 部門をたらい回しにされる現状」を再現します。続いて **②Broker あり（FB1）** を Broker ワークスペースで実行します。

## 困ったとき

| 症状 | 対処 |
|---|---|
| 窓が開かない / 接続失敗 | URL を確認してください。外部 A2A は CORS の制約があるため、dev サーバの `/proxy` 経由が必要です |
| シナリオの `$>` が流れない | MOCK モードが ON になっているか確認します（全 mock なら自動 ON）。窓名と `$>` の窓名が一致しているかも確認してください |
| 応答が出ない | 窓が「接続済み（live）」の状態か、`> 窓名` のタイムアウト内に応答が届いているかを確認します |
| 画面が古い | `⌘⇧R` でハードリロードします |
